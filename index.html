    Frequently Asked Questions

    rmdir removes empty directories only. If the directory contains any files or sub-directories the command fails.

    rm -r removes a directory and all of its contents recursively. Adding -f suppresses confirmation prompts, so use it with care — deleted files are not sent to a trash can.

    Add a shebang line as the very first line of your script, e.g. #!/bin/bash, then grant execute permission:

    chmod +x myscript.sh

    You can then run it with ./myscript.sh or by its full path.

    sudo (superuser do) lets an authorised user run a single command with elevated (root) privileges without switching accounts.

    Use it only when a command truly needs root access (installing packages, editing system config files, etc.). Running everything as root is a security risk — a typo or malicious script can damage the whole system.

    Find the process listening on a port (e.g. 8080):

    ss -tulpn | grep :8080

    Note the PID shown in the output, then kill it:

    kill -9 <PID>

    A hard link (ln target link) creates another directory entry pointing to the same inode (actual data). Deleting the original file does not remove the data — it persists as long as any hard link exists. Hard links cannot span file systems or link to directories.

    A symbolic link (ln -s target link) is a pointer that stores the path to the target. It can span file systems and point to directories, but it breaks if the target is moved or deleted.

    Use grep with the -r (recursive) and -n (line numbers) flags:

    grep -rn "search term" /path/to/directory

    Add -i for a case-insensitive search, and --include="*.py" to limit the search to specific file types.

    Use du (disk usage) with human-readable output:

    du -sh /path/to/directory

    To see the sizes of all immediate sub-directories, use:

    du -sh /path/to/directory/*

    To view overall mounted filesystem usage, use df -h.

    /dev/null is a special device file (the "black hole") that discards anything written to it and returns EOF when read.

    It is commonly used to silence unwanted output:

    command > /dev/null 2>&1 — discard both stdout and stderr.
